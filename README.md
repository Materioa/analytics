# Materio Analytics API

This is a standalone Node.js API for collecting analytics and metrics for the Materio project. It is designed to be hosted on Vercel and uses Supabase as the database.

## Features

- **Data Collection**: Accepts batched analytics events from the client.
- **CORS Enabled**: Configured to allow requests from different domains.
- **Supabase Integration**: Stores data directly into a Supabase table.

## Setup

1.  **Install Dependencies**:
    ```bash
    npm install
    ```

2.  **Environment Variables**:
    Copy `.env.example` to `.env` and fill in your Supabase credentials.
    ```bash
    cp .env.example .env
    ```

3.  **Supabase Database Setup**:
    Run the following SQL in your Supabase SQL Editor to create the required table:

    ```sql
    create table analytics_events (
      id bigint generated by default as identity primary key,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null,
      user_id uuid, 
      anonymous_id text,
      event_type text not null,
      event_data jsonb,
      client_timestamp timestamp with time zone,
      ip_address text,
      user_agent text
    );
    ```

10. **Create Recommendation RPC Function**:
    Run the SQL in `recommendations.sql` to create the function for content recommendations.

11. **Run Locally**:
    ```bash
    npm run dev
    ```

## Deployment (Vercel)

1.  Install Vercel CLI: `npm i -g vercel`
2.  Run `vercel` in this directory.
3.  Set the Environment Variables (`SUPABASE_URL`, `SUPABASE_ANON_KEY`) in the Vercel Project Settings.

## API Endpoints

### `POST /collect`

Collects a batch of analytics events.

**Payload:**

```json
{
  "userId": "optional-user-uuid",
  "anonymousId": "required-anonymous-uuid",
  "events": [
    {
      "type": "pdf_read",
      "timestamp": "2023-10-27T10:00:00Z",
      "data": {
        "pdfId": "123",
        "duration": 120,
        "country": "US" 
      }
    }
  ]
}
```

### `POST /identify`

Merges anonymous user history with a logged-in user account. Call this when a user logs in or signs up.

**Payload:**

```json
{
  "userId": "user-uuid-from-auth",
  "anonymousId": "previous-anonymous-uuid"
}
```

### `GET /stats/:userId`

Retrieves aggregated metrics for a user.

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year` (default: `today`)

**Response:**
```json
{
  "metrics": { ... },
  "top_pdfs": [ { "content_id": "123", "view_count": 5 } ],
  "streak": 3
}
```

### `GET /recommendations/:userId`

Retrieves suggested PDFs based on collaborative filtering (users who read what you read also read...).

**Query Parameters:**
- `limit`: Number of recommendations (default: 5)

### `GET /admin/users`

Retrieves active user counts (authenticated vs anonymous).

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`

### `GET /admin/features`

Retrieves usage stats for features.

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`

### `GET /admin/interactions`

Retrieves stats for interactions (scrolls, etc.).

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`

### `GET /admin/locations`

Retrieves user distribution by country (requires `country` in event data).

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`

### `GET /admin/engagement`

Retrieves total reading time across all users.

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`

### `GET /admin/top-content`

Retrieves top performing PDFs or Posts.

**Query Parameters:**
- `period`: `today`, `week`, `month`, `year`
- `type`: `pdf_read` (default) or `post_view`
